02

User Roles & Permissions

Four distinct roles, each with their own app experience. Your developer needs to build Role-Based Access Control (RBAC) from day one.

🏪
Member Business Owner
Registered SME within the chamber. Gets official digital ID card, manages their business listing, views booking requests, accesses member-only deals and tools.
🧑‍💼
Organisation Admin
Manages the chamber. Approves new member applications, issues/revokes ID cards, views analytics, manages promotions, and moderates content across the platform.
🧭
General Public / Visitor
No registration required. Can browse and discover businesses by location or category, make bookings, scan QR codes to verify member businesses. Cannot access member perks.
🏛️
Government / Partner
Read-only access to anonymised aggregated data (number of active SMEs per district, category trends). No access to individual business data. Builds institutional trust.

Core Features — Build List

These are the two core services your company is commissioned to deliver, broken into specific buildable features for your developer.

Service 1 — Digital ID Card System

F-01
Member Registration & Onboarding Flow
Application process for businesses to join the chamber and receive a verified ID.
  • Multi-step application form (business name, type, location, owner photo, ID document upload)
  • Admin review queue — approve or reject with reason note
  • Automated email/SMS notification on status change
  • Member record created in database only upon approval
F-02
Digital ID Card Generator
Auto-generates a beautiful digital ID card on member approval, viewable in-app and downloadable as PDF/image.
  • Card contains: member photo, full name, business name, member ID number, category, issue date, expiry date, and QR code
  • QR code encodes a signed JWT (member ID + expiry timestamp + signature)
  • Card rendered using HTML Canvas or react-pdf in React Native
  • Shareable as image (for WhatsApp/Telegram sharing — very important in Cambodia)
  • Card status: Active (green), Expired (orange), Revoked (red)
F-03
QR Code Authentication Scanner
Anyone (public, member, partner) can scan a member's QR code to instantly verify authenticity.
  • In-app QR scanner using Expo Camera / expo-barcode-scanner
  • Scanning shows: member name, business name, member since, status badge, and photo
  • Cryptographic verification — app validates the JWT signature without needing internet for basic check
  • Online mode: real-time database lookup to check if card has been revoked
  • Scan log stored for audit purposes (admin can view who scanned what, when)
  • Web version: unique URL (e.g. verify.app.com/member/ID123) shows same verification page
F-04
ID Card Lifecycle Management
Admins manage the full lifecycle — renewal, revocation, upgrade.
  • Cards expire after configurable period (e.g. 1 year), member gets renewal reminder 30/14/7 days before
  • Admin can revoke immediately (card shows as invalid on scan)
  • Annual renewal flow with fee tracking (for future payment integration)
  • Membership tiers (e.g. Standard / Gold / Platinum) with different card designs

Service 2 — Booking & Business Discovery

F-05
Business Directory & Marketplace Listings
Searchable, filterable directory of all member businesses — open to the public.
  • Category system (Food & Beverage, Retail, Services, Crafts, Tourism, etc.)
  • Each listing: business name, category, description, photos, location pin, operating hours, contact info, booking button
  • "Verified Member" gold badge on all chamber-approved listings
  • Search by keyword, filter by category, sort by distance / rating / newest
  • Members manage their own listing from the app
F-06
Hyperlocal Discovery (Area-Based)
The killer feature for the booking service — shows businesses near the user's current location.
  • Map view showing all nearby member businesses with pins and category icons
  • List view sorted by walking distance
  • "I'm visiting [Area Name]" mode — user selects a district/village to browse businesses there without GPS
  • Featured businesses in the area appear at top of results (premium member tier benefit)
  • Offline-cached area listings (important for rural Cambodia with patchy internet)
F-07
Booking System
Public users can book or enquire directly through the app. Businesses receive and manage bookings.
  • Two booking modes: Appointment booking (date/time slot) and Enquiry booking (message-based)
  • Business owner sets availability slots in their dashboard
  • Customer receives confirmation notification; business receives alert
  • Booking status: Pending → Confirmed → Completed / Cancelled
  • Simple chat thread per booking for communication
  • Booking history accessible to both parties
  • Optional deposit/payment note field (for future payment integration)
F-08
Promotions & Deals Engine
Members can post limited-time deals; public users can discover and claim them.
  • Deals feed — public view of all active promotions near user
  • Member posts: deal title, description, % or flat discount, validity dates, usage limit, claim code
  • Visitors scan business QR → see active deals for that business
  • Push notification: "New deal 200m away from you" (location-based trigger)
  • Admin approves deals before publishing (to maintain quality)
F-09
Admin Dashboard
Separate web panel for organisation admins to manage everything.
  • Member management: view, approve, suspend, renew members
  • ID card issuance, revocation, and audit log
  • Analytics overview: total members, active cards, bookings this month, top categories, top areas
  • Deals moderation queue
  • Announcements broadcaster (sends push notification to all members or by category)
  • Partner/government reporting: anonymised export of aggregated statistics
04

Unique Differentiators ★

These are the features that will put you ahead of any competing platform in Cambodia. None of these exist together in a single platform targeting the informal SME sector.

Your Competitive Moat

You are the only platform that combines verified identity infrastructure with hyperlocal commerce for informal businesses. Every feature below deepens that moat.

Unique
🗺️ Tourist Trail Mode
When a visitor arrives in an area, they scan a single area QR (posted at entry points, hotels, tuk-tuks) and get a curated trail of nearby verified SMEs — like a digital market guide. No app download required (web-based). This is a massive value add for tourism-adjacent businesses.
Moat
📊 Business Health Score
Each member gets a private "Business Health Score" based on: bookings completed, profile completeness, deals posted, QR scans received, and review rating. Shown only to the member. Encourages engagement and gives them a KPI they didn't have before. Scores unlock tier upgrades.
Unique
🤝 Member-to-Member Referral Network
Members can refer customers to each other within the platform — e.g., a guesthouse refers a guest to a local restaurant. The referral is tracked digitally. If a booking happens via referral, both members see it. Builds community and cross-promotion without any extra marketing cost.
Moat
🌐 Khmer-First Design
Full Khmer language UI from day one — not a translation afterthought. Khmer Unicode, right fonts, culturally appropriate UX patterns. Most competitor apps are English-primary. This alone captures the market segment that competitors cannot reach.
Unique
📶 Offline-First Core
The ID card and QR verification work fully offline. Member listings are cached for the last viewed area. Bookings queue when offline and sync when connected. Cambodia's rural internet is unreliable — apps that fail without connectivity fail their users.
Moat
🏆 Community Events Board
Admins and members can post local market events, pop-up stalls, or seasonal gatherings on a shared events calendar. Public users see events near them. This creates a reason for the public to keep checking the app — not just when they need to book, but to discover what's happening locally.
Unique
📱 WhatsApp/Telegram Deep Integration
In Cambodia, WhatsApp and Telegram are the primary communication tools. Every booking confirmation, QR scan verification, and new deal notification should be shareable directly to those platforms. Member ID cards should be shareable as a message card. Meet your users where they already are.
Moat
🔒 Trust Badge for Physical Display
Issue members a printable/laminated "Verified Chamber Member" physical sticker with QR code for display at their physical shop. This bridges digital trust into the physical world — when a customer walks past and scans the sticker on the shop door, they see the full verification. No competitor does this.

Critical Warnings for Your Developer

Security First: The QR code JWT must be signed with a secret key stored only on the server (Supabase Edge Function). Never expose the signing secret in client-side React Native code. If the secret leaks, anyone can forge verified ID cards.
Row Level Security (RLS) is mandatory: In Supabase, every table must have RLS policies. Without it, any authenticated user can read all members' data. Your developer must set RLS before going live — not after.
Build Khmer language support into the database from day one: Every user-facing text field (category names, business descriptions, notifications) should have a _km (Khmer) variant column. Adding it later is painful.
Don't use localStorage for sensitive auth tokens in the web app: Use httpOnly cookies or Supabase's built-in session management. This is a common React developer mistake with serious security implications.
GDPR/Privacy consideration: Even though Cambodia's data protection law is still developing, the QR scan log stores location data. Be transparent with users about what is logged when they scan. This protects you legally if regulations tighten.
Start with Expo (managed workflow): Your developer should not use React Native CLI bare workflow from the start. Expo managed workflow is simpler, handles push notifications, camera, and OTA updates automatically. Eject to bare only if needed later.
